# Data Model Reference

This document describes the database models and their relationships in Briefly CRM.

## Core Entities

### Organization

The root entity for multi-tenancy:

```prisma
model Organization {
    id          String   @id @default(nanoid())
    name        String
    slug        String   @unique
    logo        String?
    settings    Json?    // { timezone, locale, currency, branding }
    createdAt   DateTime @default(now())
    updatedAt   DateTime @updatedAt

    members     Member[]
    customers   Customer[]
    products    Product[]
    orders      Order[]
    // ... more relations
}
```

### User

Authentication and user management:

```prisma
model User {
    id            String       @id @default(nanoid())
    name          String
    email         String       @unique
    emailVerified Boolean      @default(false)
    image         String?
    createdAt     DateTime     @default(now())
    updatedAt     DateTime     @updatedAt

    sessions      Session[]
    accounts      Account[]
    members       Member[]
}
```

### Member

User-Organization association with roles:

```prisma
model Member {
    id             String       @id @default(nanoid())
    organizationId String
    userId         String
    role           String       @default("member") // 'root', 'admin', 'member', or custom
    createdAt      DateTime

    @@unique([userId, organizationId])
}
```

## Customer

Core e-commerce entity with RFM and lifecycle:

```prisma
model Customer {
    id    String  @id @default(nanoid())
    name  String
    phone String?
    email String?
    city    String?
    address String?

    source         CustomerSource         @default(OTHER)
    lifecycleStage CustomerLifecycleStage @default(PROSPECT)

    // E-Commerce Fields
    totalOrders          Int             @default(0)
    totalSpent           Decimal         @default(0)
    avgOrderValue        Decimal         @default(0)
    firstOrderAt         DateTime?
    lastOrderAt          DateTime?

    // RFM Scoring
    rfmScore             String?         // e.g., "555"
    rfmSegment           String?         // e.g., "CHAMPIONS"
    rfmRecency           Int?            // 1-5
    rfmFrequency         Int?            // 1-5
    rfmMonetary          Int?            // 1-5
    churnRiskScore       Float?

    // AI & Behavioral
    engagementScore       Float?
    satisfactionScore     Float?
    browsingFrequency     Float?
    cartAbandonmentRate   Float?
}
```

### Customer Enums

```prisma
enum CustomerSource {
    WEBSITE
    SOCIAL
    REFERRAL
    ORGANIC
    EMAIL
    CAMPAIGN
    OTHER
}

enum CustomerLifecycleStage {
    PROSPECT   // No Purchase Yet
    LEAD       // AI-identified potential
    ONE_TIME   // Only One Purchase
    RETURNING  // 2-4 Orders
    LOYAL      // 5+ Orders
    VIP        // Top 5% LTV
    AT_RISK    // No Purchase in more than avg.
    CHURNED    // No Purchase in more than 2x avg.
    WINBACK    // Previously churned, made new purchase
}
```

## Product

Product catalog with variants:

```prisma
model Product {
    id             String           @id @default(nanoid())
    name           String
    price          Decimal
    description    String?
    organizationId String

    externalId     String?
    sku            String?
    category       String?
    imageUrl       String?
    barcode        String?
    inventory      Int              @default(0)
    status         String           @default("active") // 'active', 'draft', 'archived'

    variants       ProductVariant[]

model ProductVariant {
    id        String   @id @default(nanoid())
    productId String
    name      String   // e.g., "Small / Blue"
    sku       String?
    price     Decimal
    inventory Int      @default(0)
    options   Json?    // e.g., {"size": "Small", "color": "Blue"}

```

## Order

Order management with fulfillment:

```prisma
model Order {
    id             String          @id @default(nanoid())
    organizationId String
    customerId     String

    shippingStatus ShippingStatus  @default(PENDING)
    paymentStatus  PaymentStatus   @default(PENDING)

    discountAmount    Decimal   @default(0)
    refundAmount      Decimal   @default(0)
    subtotal          Decimal?
    taxAmount         Decimal?
    shippingAmount    Decimal?
    totalAmount       Decimal?
    currency          String?   @default("USD")

    fulfillmentStatus String? // 'unfulfilled', 'partial', 'fulfilled'
    tags              String?

    orderItems     OrderItem[]
}

enum ShippingStatus {
    PENDING
    PROCESSING
    SHIPPED
    DELIVERED
    CANCELLED
}

enum PaymentStatus {
    PENDING
    PAID
    FAILED
    REFUNDED
}
```

## Integrations

External service connections:

```prisma

    id           String       @id @default(nanoid())
    orgId        String
```

## Indexes

Key indexes for performance:

| Model      | Index                          | Purpose                |
| ---------- | ------------------------------ | ---------------------- |
| Customer   | `[organizationId]`             | Org-scoped queries     |
| Customer   | `[email]`                      | Email lookups          |
| Customer   | `[lifecycleStage]`             | Lifecycle filtering    |
| Customer   | `[rfmSegment]`                 | RFM segment filtering  |
| Order      | `[organizationId, customerId]` | Org + customer queries |
| Product    | `[organizationId, externalId]` | External ID lookups    |
| ImportJob  | `[organizationId, status]`     | Job status queries     |
| WebhookLog | `[integrationId, topic]`       | Webhook filtering      |

## Best Practices

### Queries

Always filter by `organizationId` for multi-tenant queries:

```typescript
const customers = await prisma.customer.findMany({
    where: {
        organizationId: user.organizationId
    }
});
```

### Pagination

Use `take` and `skip` for pagination:

```typescript
const customers = await prisma.customer.findMany({
    where: { organizationId },
    take: 20,
    skip: (page - 1) * 20
});
```

### Relations

Include related data with `include`:

```typescript
const orders = await prisma.order.findMany({
    where: { organizationId },
    include: {
        customer: true,
        orderItems: {
            include: { product: true }
        }
    }
});
```
