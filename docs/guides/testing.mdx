# Testing

Briefly CRM uses Bun's built-in test runner with supertest for HTTP testing.

<Callout title="Test Database">
    Tests require a separate database. Set `DATABASE_URL` to a test database
    connection.
</Callout>

## Running Tests

### All Tests

```bash
bun test
```

### Single Test File

```bash
bun test src/api/customers/customer.test.ts
```

### Test Pattern

```bash
bun test --test-name-pattern "pattern"
```

## Test Structure

Tests follow the same structure as source files:

```
src/
├── api/
│   └── customers/
│       ├── customer.controller.ts
│       ├── customer.test.ts    # Tests for controller
│       ├── customer.service.ts
│       └── customer.service.test.ts
```

## Test Patterns

### HTTP Tests

Use supertest for endpoint testing:

```typescript
import { describe, it, expect, before, after } from 'bun:test';
import request from 'supertest';
import { app } from '../../app.js';

describe('GET /api/customers', () => {
    it('returns customers list', async () => {
        const response = await request(app)
            .get('/api/customers')
            .set('Authorization', `Bearer ${token}`);

        expect(response.status).toBe(200);
        expect(response.body.data).toBeDefined();
    });
});
```

### Service Tests

Test business logic in isolation:

```typescript
import { describe, it, expect } from 'bun:test';
import { calculateRFM } from './rfm.service.js';

describe('calculateRFM', () => {
    it('calculates correct score for active customer', () => {
        const result = calculateRFM({
            lastOrderAt: new Date('2024-01-01'),
            orderCount: 10,
            totalSpent: 5000
        });

        expect(result.score).toBeGreaterThan(0);
    });
});
```

### Database Tests

Use `beforeAll` and `afterAll` for setup:

```typescript
import { describe, it, expect, beforeAll, afterAll } from 'bun:test';
import prisma from '../config/prisma.config.js';

describe('Customer CRUD', () => {
    let testCustomer: any;

    beforeAll(async () => {
        testCustomer = await prisma.customer.create({
            data: {
                email: 'test@example.com',
                name: 'Test User',
                organizationId: 'org_test'
            }
        });
    });

    afterAll(async () => {
        await prisma.customer.delete({ where: { id: testCustomer.id } });
    });

    it('creates customer', () => {
        expect(testCustomer.email).toBe('test@example.com');
    });
});
```

## Fixtures

### Test Users

Create test users with proper organization associations:

```typescript
async function createTestUser(organizationId: string) {
    return prisma.user.create({
        data: {
            email: `test-${Date.now()}@example.com`,
            name: 'Test User',
            organizationId
        }
    });
}
```

### Test Data Cleanup

Clean up after tests:

```typescript
afterAll(async () => {
    await prisma.customer.deleteMany({
        where: { organizationId: 'org_test' }
    });
});
```

## Assertions

### Response Assertions

```typescript
expect(response.status).toBe(200);
expect(response.body.data).toBeDefined();
expect(response.body.data.length).toBeGreaterThan(0);
```

### Error Assertions

```typescript
expect(response.status).toBe(400);
expect(response.body.message).toContain('validation');
```

### Array Assertions

```typescript
expect(customers).toHaveLength(5);
expect(customers[0]).toMatchObject({
    id: expect.any(String),
    email: expect.any(String)
});
```

## Mocking

### External Services

Mock external APIs when needed:

```typescript
import { mockFn } from 'bun:test';

mockFn(fetch, async () => {
    return new Response(JSON.stringify({ ok: true }));
});
```

## Integration Tests

### Database Setup

Integration tests require a test database:

```bash
# Create test database
createdb briefly_test

# Run tests
DATABASE_URL=postgresql://localhost/briefly_test bun test
```

### Auth Tests

Test authentication flow:

```typescript
describe('Authentication', () => {
    it('signs in with email/password', async () => {
        const response = await request(app)
            .post('/api/auth/sign-in/email')
            .send({ email: 'test@example.com', password: 'password123' });

        expect(response.status).toBe(200);
        expect(response.body.data.session.token).toBeDefined();
    });
});
```

## Best Practices

### Test Naming

Use descriptive test names:

```typescript
it('returns 400 when email is invalid', () => { ... });
it('creates customer with valid data', () => { ... });
```

### AAA Pattern

Follow Arrange-Act-Assert:

```typescript
it('creates customer', async () => {
    // Arrange
    const customerData = { email: 'test@example.com', name: 'Test' };

    // Act
    const response = await request(app)
        .post('/api/customers')
        .send(customerData);

    // Assert
    expect(response.status).toBe(201);
});
```

### Independent Tests

Each test should be independent:

```typescript
beforeEach(async () => {
    // Clean state before each test
    await prisma.customer.deleteMany();
});
```

## Troubleshooting

### Tests Hang

Ensure database connections are closed:

```typescript
afterAll(async () => {
    await prisma.$disconnect();
});
```

### Flaky Tests

Avoid timing-dependent tests. Use explicit waits or retries when needed.

### Database Conflicts

Use unique identifiers for test data:

```typescript
const email = `test-${Date.now()}@example.com`;
```
